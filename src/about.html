Om webbplatsen

I denna webbplats:

erbjuds det möjligheten av att lagra och visa besökarnas arbetserfarenheter. I ”Lägg till erfarenhet” sidan kan man fylla i formuläret med uppgifter om en arbetserfarenhet. Uppgifterna lagras då i databasen. På landningssidan visas det alla lagrade arbetserfarenheter som finns i databasen.

Webbplatsen använder en REST API där GET anrop till API:t hämtar alla lagrade uppgifter från databasen, POST anrop skickar in uppgifter att lagra i databasen, DELETE anrop med specifikt ID raderar en rad i databasen och PUT anrop uppdaterar en viss rad i databasen med ett angivet ID.

Databasen är en NoSQL databas som är byggd på MongoDB DBMS, databasen har en collection (workexperience) som följer ett schema som är uppsatt av Mongoose. Denna Collection innehåller 5 fields (ID, companyname, jobtitle, location, startdate och enddate). ID är ett 12 byte långt värde som generas per automatik av MongoDB. Companyname, jobtitle och location är textsträngar, startdate och enddate är datumobjekt. Samtliga kolumner måste innehålla något värde dvs inte får vara NULL förutom enddate då det kan vara så att man vill registrera info om ett pågående jobb.

Att bygga upp ett API och använda det för att kommunicera med klienten erbjuder betydligt mycket mer flexibilitet än att rendera html data från servern och hantera json data i html med ejs. På det sättet, med API, servern hanterar data och kommunikation med databasen och skickar resultatet med API till klienten, där med hjälp av javascript kan json data bearbetas, struktureras och dynamiskt visas i klientens html sidorna.
